# ADR 0014 — Next.js + FastAPI + Postgres + Hetzner US single VPS

**Status:** Accepted
**Date:** 2026-06-14
**Depends on:** 0005, 0009, 0010, 0013
**Consumed by:** 0015

## Context

PRD `tech_stack` was a placeholder list ("React or Next.js," "FastAPI or Node.js," "Postgres," "Redis," etc.). ADR 0009 pinned ASR/TTS/LLM. Remaining open choices (frontend, backend, DB hosting, background work, observability, charts) feed directly into cost envelope, M1 scope, US data residency for JCPS, and pilot defensibility.

User has no local compute (ADR 0009 constraint). Stack must be cloud-hostable on a single low-cost VPS.

## Decision

| Layer | Choice |
|-------|--------|
| **Frontend** | Next.js 15 App Router + Tailwind + shadcn (copy-into-repo) + Recharts + `next-intl` (es + en) + TanStack Query; deployed on Vercel free tier |
| **Backend** | Python 3.12 + FastAPI + SQLModel + Alembic + httpx (async); uvicorn workers in prod |
| **Database** | Postgres 16 + pgvector (HNSW indexes); self-hosted on VPS for MVP; migration to Supabase Pro at M3+ when DB > 4GB or scale demands |
| **Cache** | Redis 7 (in-VPS): magic-link TTL, Anthropic prompt cache mirror, session state |
| **Queue** | RQ (Redis Queue) for async jobs + APScheduler for cron-style recurring; Celery deferred to M3+ |
| **Hosting** | **Hetzner Cloud CX22, US-Ashburn** — 2 vCPU / 4GB RAM / 40GB SSD, ~$5/mo; FastAPI + Postgres + Redis + workers + nginx co-located |
| **Storage** | Cloudflare R2 (10GB free tier) — audio + nightly `pg_dump` backups; 14-day retention |
| **Email** | Resend (free tier 100/day, 3k/mo) — magic links + notifications |
| **SMS** | Twilio (~$0.0075/msg US) — SMS magic links (M2+) |
| **DNS + CDN + TLS** | Cloudflare (free) |
| **LLM** | Anthropic Claude (Haiku 4.5 hot path, Sonnet 4.6 grader) per ADR 0009 |
| **ASR** | Deepgram Nova-2 streaming per ADR 0009 |
| **TTS** | OpenAI `tts-1` streaming per ADR 0009 |
| **Charts** | Recharts (learner / instructor UI) + Plotly (coordinator PDF reports only) |
| **Observability** | Sentry (free tier 5k events/mo) + structured JSON logs (logrotate) + in-DB `cost_events` for unit economics + `voice_turn_metrics` for latency |
| **Testing** | pytest (backend) + Playwright (frontend critical paths) |
| **CI/CD** | GitHub Actions (lint + test + lesson YAML lint + DB migration check + deploy via SSH on merge) |

**Why Python over Node.js for backend**: ADR 0009d's M2 path (SpeechAce) and M3+ path (forced phoneme alignment via charsiu / Montreal Forced Aligner) only exist in the Python ecosystem. Choosing Node now means rewriting later. FastAPI's async voice streaming is good enough for ADR 0009's latency budget.

**Why single-VPS topology**: pilot scale doesn't warrant a parallel observability stack or separate worker fleet. Migration paths (Supabase Pro, second VPS) are explicit and gated on signal.

**US data residency**: Hetzner US locations (Ashburn VA, Hillsboro OR) satisfy district preferences for FERPA / KY SB 216 alignment. **Non-negotiable** for the JCPS pilot.

**Fixed infra at pilot scale**: ~$15/mo (VPS + Twilio + free tiers). Plus cloud APIs from ADR 0009 (~$160–215/mo at 20-learner pilot). **Total ~$175–230/mo.**

**Scale ceiling**: ~50–80 active learners on this topology. Past that: VPS upsize (CX32 $10/mo), Supabase Pro migration, second VPS for workers. M3+ problem.

## Consequences

- Concrete repo structure:
  ```
  /frontend            Next.js App Router
  /backend             FastAPI app (engine, tutor, mastery, ingestion, voice)
  /workers             RQ workers + APScheduler
  /content             Lesson YAML + rubrics
  /data                Corpus (built before grilling)
  /docs                PRD, roadmap, ADRs, runbooks, legal
  /tests
  /scripts             pg_dump cron, cost rollup, lesson lint
  /infra               Hetzner provisioning, nginx, systemd
  ```
- Secrets per-env in `.env` (gitignored); production secrets in VPS systemd env; rotation runbook.
- Nightly `pg_dump` to R2 with 14-day retention; restorable to fresh VPS in <30 minutes.
- pgvector HNSW indexes on chunk embeddings.
- OpenAPI spec generated by FastAPI; consumed by Next.js via openapi-typescript-codegen.
- Three RQ workers: default, engine, apscheduler.
- Observability is **in-app** (Sentry + DB tables), not a parallel stack. Cost data must be queryable from coordinator dashboard.

## Alternatives considered

- **Node.js + TypeScript backend** — rejected for ADR 0009d's forced-aligner roadmap.
- **Supabase managed Postgres at MVP** — rejected for cost; pilot scale doesn't justify $25/mo.
- **Neon (serverless Postgres)** — viable; offers burst capacity, but self-host is cheaper at pilot scale.
- **Celery + Redis from day one** — rejected. RQ + APScheduler fit the 4GB VPS; Celery is overkill until scale demands.
- **Plain React (Vite)** — rejected. Loses i18n + SSR + App Router benefits; rebuilds more by hand.
- **DigitalOcean / Fly.io** — viable. Hetzner CX22 is cheapest at the spec; US-Ashburn region available.
- **Prometheus + Grafana** — deferred. Pilot scale doesn't justify a parallel metrics pipeline; in-DB metrics suffice.
